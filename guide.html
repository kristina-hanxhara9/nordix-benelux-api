⚠ Netherlands & Luxembourg — specific issues & workarounds

These two countries have the most friction for data science work. Here are the exact problems and the best workarounds for each.

KVK
Cannot filter by SBI code. No ?sbi= parameter exists on any KVK endpoint. You cannot say "give me all IT companies".
Free bulk has no names. The free open dataset strips company names and KVK numbers for GDPR reasons. SBI codes are there but you can't attach them to a named company for free in bulk.
Basisprofiel costs €0.02/call. To get SBI code + employees + full address you pay per company on top of the €6.40/month base.
Zoeken is exact-match only. "koophand" returns zero results. "koophandel" returns 1,000. No fuzzy search.
BV/NV only in free dataset. Sole traders, foundations, partnerships not included.
LBR
CAPTCHA since August 2025. The portal actively blocks automated access. No scraping possible on the free portal.
No bulk download at any tier. Even if you pay for the professional API, there is no "download all 168K companies" endpoint.
No NACE codes for free. They exist (assigned monthly by STATEC) but only accessible via the paid professional API.
Professional API: unknown price. No public pricing page. Must email helpdesk@lbr.lu. Enterprise contracts only.
French/German only. All documents, portal interface, and purpose text are in French or German. No English data.

Workarounds

For name + SBI without paying €0.02/call: Use Zoeken (€6.40/mo key, €0/call) to get KVK number → query free open dataset with that KVK number to get SBI code. Join key = KVK number. Total extra cost = €0.

For sector sweeps (all companies of type X): KVK cannot do this. Use Belgium free CSV (filter activity.csv on NaceCode) or Norway free API (?naeringskode=) instead.

For SBI filtering in Netherlands specifically: Download the anonymised free bulk dataset, filter by SBI code offline, get a list of KVK numbers, then call Zoeken for each to get the company name. Works but slow for large sets.

Workarounds

For free data at scale: Not possible. Luxembourg is not suitable for bulk free data pipelines. For individual company lookups the portal works fine manually.

For NACE codes free: Not possible directly. Workaround: download articles of association PDFs manually → extract purpose text → classify with keyword rules or LLM. But manual download only (no automation due to CAPTCHA).

For bulk structured data: Pay for professional API (email helpdesk@lbr.lu) or buy from a third-party aggregator like OpenCorporates or North Data which have already licensed the data.

For historical filings: RESA covers 2016→today. Mémorial C covers pre-2016. Both free, both on lbr.lu, both PDFs, both manual only.

Your realistic options for Luxembourg data:

Option 1  Manual lookups only — for specific companies you already know by name. Free, works, slow.

Option 2  Professional API — email helpdesk@lbr.lu for pricing. Gives structured JSON, NACE codes, shareholders. Requires LuxTrust certificate + signed agreement.

Option 3  Third-party data provider — OpenCorporates, Kyckr, or North Data have licensed Luxembourg data and sell API access. Often more practical than dealing with LBR directly.

Not recommended  Building a scraper — CAPTCHA will block it. LBR explicitly wants machine users on the paid API, not scraping the portal.
